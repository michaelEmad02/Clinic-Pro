## Architecture Pattern — Clean Architecture

The project follows **Clean Architecture** with strict layer separation.
Each feature is fully self-contained with its own 3 layers.

```
┌─────────────────────────────────────────────────────────┐
│                  Presentation Layer                     │
│         (UI Screens + SubWidgets + Cubit/Bloc)          │
├─────────────────────────────────────────────────────────┤
│                    Domain Layer                         │
│          (Entities + UseCases + Repo Interfaces)        │
├─────────────────────────────────────────────────────────┤
│                     Data Layer                          │
│        (Models + DataSources + Repo Implementations)    │
├─────────────────────────────────────────────────────────┤
│                   External Sources                      │
│              (Supabase / Mock Services)                 │
└─────────────────────────────────────────────────────────┘
```

**Dependency Rule:** outer layers depend on inner layers — never the reverse.

```
Presentation → Domain ← Data
                ↑
              Core
```

## Service Layer Architecture (Data Flow)

This is the critical data flow from UI to Supabase and back:

```
UI (Cubit/Bloc)
      ↓ calls
UseCase (Domain)
      ↓ calls
RepositoryImpl (Data)
      ↓ calls
DataSource (Data)
      ↓ calls
ICloudService (Core/Services)
      ↓
  [Real]  SupabaseService     → Supabase SDK → returns raw Map<String,dynamic>
  [Mock]  MockCloudService → returns static raw Map<String,dynamic>
```

### Key Responsibility Separation

| Layer | Responsibility |
|-------|---------------|
| `ICloudService` / `SupabaseService` | Communicates with Supabase SDK. Returns **raw data only** (`Map<String,dynamic>` or `List<Map>`). No parsing. |
| `DataSource` | Calls the service, **converts raw data → Model**. Handles exceptions. |
| `RepositoryImpl` | Calls DataSource, wraps result in `Either<Failure, Entity>` |
| `UseCase` | Single operation on Repository. Contains business validation logic. |
| `Cubit/Bloc` | Calls UseCase, emits States to UI |
| `UI Screen` | Renders state. Delegates sub-parts to SubWidgets. |

## Dependency Injection — getIt + injectable

```dart
// Registration via @injectable annotations
@module
abstract class AppModule {
  @lazySingleton
  SupabaseClient get supabaseClient => Supabase.instance.client;
}

// Feature-level example
@LazySingleton(as: IAppointmentRepository)
class AppointmentRepositoryImpl implements IAppointmentRepository { ... }

// Switch between real and mock by changing annotation target
@LazySingleton(as: ICloudService)
class MockCloudService implements ICloudService { ... }
// ↑ change to SupabaseService when backend is ready
```

Run `flutter pub run build_runner build` to regenerate `injection_container.config.dart`.

## State Management Rules

| Scenario | Use |
|----------|-----|
| Simple toggle, loading flag, single screen state | **Cubit** |
| Complex events, multiple event types, event transformations | **Bloc** |
| Never | `setState` (except forced edge cases like animations) |

## Navigation — GoRouter

```dart
// Role-based redirect on startup
redirect: (context, state) {
  final userType = getIt<AuthCubit>().state.userType;
  return switch(userType) {
    UserType.owner     => '/owner/dashboard',
    UserType.doctor    => '/doctor/dashboard',
    UserType.secretary => '/secretary/dashboard',
    _                  => '/login',
  };
}
```

## Responsive Strategy

```dart
// core/utils/responsive_helper.dart
class ResponsiveHelper {
  static bool isMobile(BuildContext context)  => MediaQuery.of(context).size.width < 600;
  static bool isTablet(BuildContext context)  => MediaQuery.of(context).size.width >= 600
                                              && MediaQuery.of(context).size.width < 900;
  static bool isDesktop(BuildContext context) => MediaQuery.of(context).size.width >= 900;
}

// Usage in widgets
if (ResponsiveHelper.isTablet(context))
  TwoColumnLayout(...)
else
  SingleColumnLayout(...)
```
